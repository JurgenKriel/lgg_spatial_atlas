# Plan 02-02 — Fuse pt2 layers into AnnData

## Objective

Using the join decided in 02-01, assemble one AnnData `h5ad` per pt2 sample
carrying gene expression, metabolite features (flagged distinctly), cell type,
niche, centroids, and colour palettes — in the exact conventions Phase 1's
`PIPELINE_CONTRACT.md` documented for the webatlas h5ad ingestor.

## Context

- Inputs are the scratch dumps + `JOIN_KEY.md` from 02-01.
- Target conventions from `venture_atlas/webatlas/PIPELINE_CONTRACT.md` (Phase 1):
  the exact `obsm` spatial key, obsSets mechanism, and feature-type/second-matrix
  support determine whether metabolites live as extra `var` rows with
  `var['feature_type']` or as a second matrix. Follow whichever the contract says.
- Zarr discipline: if the pipeline (Phase 3) writes v3, that's handled in Phase 3;
  the h5ad here is standard AnnData.

## Tasks

<task id="1" type="execute">
<title>Build the per-sample AnnData</title>
<read_first>
- venture_atlas/build/JOIN_KEY.md
- venture_atlas/webatlas/PIPELINE_CONTRACT.md
- venture_atlas/build/pt2_centroids_summary.json
</read_first>
<action>
Write `venture_atlas/build/build_pt2_anndata.py` (run under `spatialdata_env_2`).
For each pt2 sample:
  - Load the centroids zarr `cell_annotations` as the obs anchor (defines cell set + order).
  - Attach gene matrix aligned to those cells via the 02-01 join → `adata.X`, `var` = genes with `feature_type='gene'`.
  - Attach metabolite matrix aligned to those cells → append as extra `var` rows
    with `feature_type='metabolite'` (or a separate matrix / `obsm`, per PIPELINE_CONTRACT).
    Cells lacking a metabolite match get NaN/0 per the contract's missing-value rule.
  - Set `obs['cell_type']` = `Anno`, `obs['niche']` = `niche` (as pandas categoricals).
  - Set `obsm['spatial']` = the x/y centroids (per-z-plane handled per contract: one
    h5ad per (sample,z-plane) OR a z column — choose per contract, default per-z-plane).
  - Copy `Anno_colors` / `niche_colors` palettes into `adata.uns`.
  - Write `h5ad` to `/vast/scratch/users/kriel.j/atlas_pt2/h5ad/<sample>[_z<z>].h5ad`.
</action>
<acceptance_criteria>
- Running the script produces at least one `<sample>*.h5ad` in the h5ad outdir.
- `python -c "import anndata; a=anndata.read_h5ad(<f>); assert 'cell_type' in a.obs and 'niche' in a.obs and 'spatial' in a.obsm and 'feature_type' in a.var.columns"` exits 0.
- `a.var['feature_type']` contains both 'gene' and 'metabolite' values.
</acceptance_criteria>
</task>

<task id="2" type="execute">
<title>Validate the AnnData against the contract + join thresholds</title>
<read_first>
- venture_atlas/build/build_pt2_anndata.py
- venture_atlas/build/JOIN_KEY.md
</read_first>
<action>
Write `venture_atlas/build/validate_pt2_anndata.py`. For each written h5ad assert:
  - obs cell count equals the centroid cell count for that sample/z-plane.
  - fraction of cells with a non-missing metabolite value equals the matched
    fraction recorded in JOIN_KEY.md (within tolerance).
  - `obsm['spatial']` has 2 columns and finite coordinate ranges matching the
    centroids summary.
  - the `obsm` spatial key name and obs categorical names match what
    PIPELINE_CONTRACT.md says the ingestor reads.
  - `uns` holds the colour palettes with one colour per category.
Exit non-zero on any failure.
</action>
<acceptance_criteria>
- `validate_pt2_anndata.py` exits 0 on the built h5ad(s).
- It prints per-sample: cell count, gene count, metabolite count, metabolite-coverage fraction.
</acceptance_criteria>
</task>
