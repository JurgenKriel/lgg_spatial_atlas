# Plan 02-01 ÔÇö Resolve the join key and extract pt2 layers

## Objective

Before fusing anything, prove HOW pt2 cells from the three sources map to each
other, and dump each source's pt2 slice to intermediate files with a common
`cell_id`. This de-risks the whole atlas: if the keys don't line up, everything
downstream is wrong.

## Context

- Sources (all verified to exist 2026-07-20):
  - Genes: `/stornext/Bioinf/data/lab_brain_cancer/projects/tme_spatial/transcriptomics/venture/ST/data/processed/ven_all_250217.rds` (Seurat, full cohort ÔÇö subset to pt2).
  - Cell type/niche/centroids: `venture_ST/centroids/ven2_centroids.zarr` (and any pt2 sub-sample zarrs ÔÇö confirm the sample naming for pt2).
  - Metabolites: `venture_pt2/integrated/integrated_data_250925.rds` (SpaMTP/Seurat, aligned to ST).
- Centroids `cell_annotations` obs has: `cell_id`, `instance_id`, `sample_id`, `Anno`, `niche`, `x/y/z_centroid`, `z_layer`.
- R env: `module load R/4.5.3`, `LD_LIBRARY_PATH=cellpose_env/lib`,
  `R_LIBS=R_libraries:Venture_lib/lib/R/library` (per project memory). SpaMTP via `SpaMTP_env`.

<action>

  (a) direct key join if barcodes/cell_ids match (preferred), or
  (b) spatial nearest-neighbour on centroids with a distance threshold if the
      metabolomics uses a different key ÔÇö document the threshold and the fraction

Record the chosen key, the matched-fraction number, and the minimum acceptable
threshold (propose Ôëą0.8 of centroid cells; flag for user if lower).
</action>

- It states a pass/fail against the documented threshold.
</acceptance_criteria>
</task>
